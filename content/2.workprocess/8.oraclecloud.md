# ขั้นตอน Deploy ระบบบน Oracle Cloud

## 1. สร้าง Instance บน Oracle Cloud

เข้า Oracle Cloud แล้วสร้าง Instance สำหรับใช้เป็น Host ของระบบ จากนั้น Download ไฟล์ Private Key เก็บไว้ทันที เพราะใช้สำหรับ SSH เข้าเครื่อง

> หากไม่มี Private Key จะไม่สามารถเข้า Host ได้

ตัวอย่าง Private Key

```bash
/home/koranit/Downloads/instance2/ssh-key-2026-05-04.key
```

ตัวอย่าง Public IP

```bash
161.118.204.74
```

---

## 2. ตั้งค่า VCN และ Subnet

ตั้งค่า VCN และ Subnet ให้ Instance สามารถเชื่อมต่อ Internet ได้ และมี Public IP สำหรับเข้าใช้งานจากภายนอก

---

## 3. SSH เข้า Instance

กำหนดสิทธิ์ให้ Private Key

```bash
chmod 400 /home/koranit/Downloads/instance2/ssh-key-2026-05-04.key
```

SSH เข้าเครื่อง Oracle Cloud

```bash
ssh -i /home/koranit/Downloads/instance2/ssh-key-2026-05-04.key ubuntu@161.118.204.74
```

การ SSH คือการเข้าใช้งานเครื่อง Server อีกเครื่องผ่าน Command Line

## 7. Deploy ระบบ


```bash

./script/deploy.sh
```

และเช็ค pm2

## 8. Restore Database

ติดตั้ง MariaDB

```bash
sudo apt install -y mariadb-server
sudo systemctl start mariadb
sudo systemctl enable mariadb
```

เข้า MariaDB

```bash
sudo mysql
```

สร้าง Database

```sql
CREATE DATABASE payroll CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
EXIT;
```

ถ้าไฟล์ SQL อยู่บน Host แล้ว

```bash
sudo mysql payroll < payroll.sql
```

ถ้าไฟล์ SQL อยู่ในเครื่อง Local ให้ส่งขึ้น Host ก่อน

```bash
scp -i /home/koranit/Downloads/instance2/ssh-key-2026-05-04.key payroll.sql ubuntu@161.118.204.74:/home/ubuntu/
```

แล้ว Restore บน Host

```bash
sudo mysql payroll < /home/ubuntu/payroll.sql
```
หรือ ทำผ่าน filezilla

## 9. เปิด Port ใน Oracle Security List

เปิด Port `8080` สำหรับ `kxreport`

```
Source CIDR: 0.0.0.0/0
Protocol: TCP
Source Port Range: เว้นว่าง
Destination Port Range: 8080
Description: kxreport service
```

เปิด Port `80` สำหรับ `kh-attendance frontend`

```
Source CIDR: 0.0.0.0/0
Protocol: TCP
Source Port Range: เว้นว่าง
Destination Port Range: 80
Description: kh-attendance frontend
```

> Source Port Range ให้เว้นว่าง

## หมายเหตุสำคัญ

- ต้อง Download Private Key ตอนสร้าง Instance
- ก่อน SSH ต้องใช้ `chmod 400` กับไฟล์ Key
- เปิด Port `8080` สำหรับ `kxreport`
- เปิด Port `80` สำหรับ `kh-attendance frontend`
- Source Port Range ใน Security List ให้เว้นว่าง
- Oracle Cloud ไม่ควรเปิดใช้งานผ่าน Port `3000` โดยตรง
- ควรใช้ Nginx ให้เข้าเว็บผ่าน Port `80`
- ตั้ง Timezone เป็น `Asia/Bangkok`
- หากอัปเดต Code ให้ใช้ `git pull` แล้ว Build / Deploy ใหม่
- หากเปลี่ยน Report ให้รัน `copyreporthost.sh`
