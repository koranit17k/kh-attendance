# Frontend Workflow

โปรเจกต์เรียนรู้การบันทึกข้อมูลจากหน้า Frontend เข้าสู่ Database และนำข้อมูลกลับมาใช้แสดงผล
> **Git:** [kh-attendance](https://github.com/koranit17k/kh-attendance.git)

---

### 📦 1. การติดตั้งและเตรียมโครงการ (Setup)
*   **การติดตั้ง:** ใช้คำสั่ง `pnpm install vue, nuxt` เพื่อเริ่มโปรเจกต์
*   **การรันระบบ:** ใช้คำสั่ง `pnpm dev` เพื่อรัน Local และทดสอบหน้าเว็บ
*   **การตั้งค่า:** อย่าลืมเช็คไฟล์คอนฟิกหลัก:
    - `nuxt.config.ts`
    - `package.json`
    - `tsconfig.json`

---

### 🎨 2. การพัฒนาส่วนหน้าจอ (Components)
เป็นการสร้างส่วนประกอบต่างๆ เพื่อนำมาประกอบร่างในหน้าเว็บหลัก:
*   **สร้าง Component:** ออกแบบและเขียนโค้ดแยกส่วน (เช่น ปุ่ม, ตาราง, กราฟ)
*   **นำไปใช้:** นำ Component ที่สร้างไว้ไปใส่ในหน้าหลัก (Pages)
*   **Component สำคัญ:**
    - **Calendar:** ปฏิทินเช็ควันที่พนักงานมาทำงาน
    - **Attendance Sum:** KPI Card สรุปยอด มาทำงานเต็มวัน/ครึ่งวัน/ขาด

---

### ⚙️ 3. การจัดการข้อมูลและ API (Backend & DB)
เน้นการจัดการข้อมูลหลังบ้านและการเชื่อมต่อกับฐานข้อมูล:
*   **Database:** สร้าง Table และ api ชื่อ `counter` สำหรับทดสอบการนับข้อมูล
*   **API Layer (โฟลเดอร์ server):**
    - **api:** ตัวกลางรับส่งข้อมูลระหว่างหน้าเว็บกับ DB
    - **middleware:** ตัวนับ Session และจัดการ Cookie ในเว็บ
    - **utils:** ส่วนเชื่อมต่อกับ Database โดยตรง
*   **การรายงานผล:** สร้าง **Call Report** สำหรับเรียกใช้รายงานผ่าน API

---

> [!TIP]
> **Flow การตรวจสอบ:** หลังรันระบบแล้ว ให้ลองเข้าไปใช้งานหน้าเว็บ แล้วกลับไปเช็คข้อมูลใน Database (Table `counter`) เพื่อดูว่าข้อมูลถูกบันทึกจริงหรือไม่
