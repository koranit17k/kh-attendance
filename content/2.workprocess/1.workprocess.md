## 🏁 Phase 1: การวางแผนและเตรียมการ
1. **กำหนดขอบเขตโครงการ:** ระบุหัวข้อและรายละเอียดระบบบันทึกเวลาเข้า-ออกงาน
2. **กำหนดเงื่อนไขทางธุรกิจ:** ระบุเกณฑ์เวลาเข้างาน-เลิกงาน และเงื่อนไขการคิดเวลาทำงาน
3. **เตรียมโครงสร้างพื้นฐาน:**
    - ออกแบบและสร้าง Database
    - กำหนดสิทธิ์การใช้งาน User ในระบบฐานข้อมูล

---

## 📊 Phase 2: การจัดการข้อมูล (Data Management)
4. **วิเคราะห์กลุ่มข้อมูล:** จำแนกกลุ่มข้อมูลตามเงื่อนไข (เพื่อแสดงผลสีในระบบ)
    - 🟢 **เคสสีเขียว:** คำนวณเวลาทำงานปกติได้
    - 🟡 **เคสสีเหลือง:** ทำงานครึ่งวัน หรือเข้า/ออกไม่ครบตามเงื่อนไข
    - 🔴 **เคสสีแดง:** ข้อมูลผิดพลาด ไม่สามารถคำนวณเวลาได้
5. **ตรวจสอบความถูกต้อง:** ตรวจทานความสมบูรณ์ของข้อมูลดิบ
6. **จัดการข้อมูลพนักงาน:**
    - เพิ่มรายชื่อพนักงานใหม่เข้าสู่ `employee` table
    - อัปเดตข้อมูลส่วนตัว เช่น รหัสพนักงาน, `timecode`
    - จัดการสถานะพนักงานลาออก (ตั้งค่า `timecode` เป็น 0)

---

## ⚙️ Phase 3: การพัฒนาระบบคำนวณ (Logic & Automation)
7. **เขียน Logic การคำนวณ:** พัฒนาสูตรการคิดเวลางาน, สาย (Late) และ OT
8. **สร้างเครื่องมือจัดการข้อมูล:**
    - สร้าง `attendance` table สำหรับใช้เป็นพื้นที่ปรับแก้ข้อมูลที่ผ่านการกรองแล้ว
    - พัฒนา **Stored Procedure** สำหรับประมวลผลข้อมูลอัตโนมัติ
9. **ตั้งค่าระบบอัตโนมัติ (Event):**
    - สร้าง **Event Scheduler** เพื่อให้ Procedure ทำงานอัตโนมัติ (Auto-run) ทุกวัน

---

## 🧪 Phase 4: การทดสอบและประกันคุณภาพ (QA)
10. **กำหนด Scenario:** จัดทำ [Test Case Scenarios](3.exampledata.md) เพื่อรองรับรูปแบบข้อมูลที่หลากหลาย
11. **ทดสอบด้วย Unit Test:**
    - ใช้ **Vitest** ในการทดสอบความถูกต้องของ View `vAttendance`
    - ตรวจสอบความแม่นยำของสูตรการคำนวณเวลางาน, สาย และ OT
12. **เกณฑ์การผ่าน:** ผลการทดสอบต้องเป็น **100% Pass** เท่านั้น [ดูรายละเอียด Vitest](4.vitest.md)

---

## 🖥️ Phase 5: การแสดงผลและรายงาน (Reporting & UI)
13. **สร้างระบบรายงาน:** จัดทำ Jasper Report สำหรับพิมพ์สรุปผล
14. **พัฒนา API สำหรับรายงาน:** เชื่อมต่อระบบเรียกดู Report ผ่าน API [Workflow Report](5.workflowreport.md)
15. **พัฒนา Frontend:** สร้างส่วนติดต่อผู้ใช้ (UI) สำหรับบันทึกและจัดการข้อมูลโครงการ

---

## 💡 ข้อแนะนำ (Notes)

- การสร้าง procedure ช่วยลดเวลาคำนวน และ ช่วยในการลดความซับซ้อนของโค้ด
- ตอนสร้าง event 
    - การ insert,update database อัตโนมัติ ผ่าน query. ข้อมูลเกี่ยวกับ เช่น รหัสสแกน,เวลา,วันที่พนักงานขาด รวมไปถึงการคำนวนเวลา 
    ```sql 
        sudo mysql
            GRANT EVENT ON *.* TO 'koranit'@'%'; -- in terminal root , 
            SET GLOBAL event_scheduler = ON; -- in terminal root after grant event  
    ```
    - ต้องระวังเรื่องการ restart database เราต้องเข้าไปปรับใน config เพื่อไม่ให้มันปรับ OFF event โดยไม่ตั้งใจ (event ไม่ค่อยถูก support ในโปรแกรม ต้องตั้งค่าเพิ่ม)
    - ไฟล์ config อยู่ที่ /etc/mysql/mariadb.conf.d/50-server.cnf
    - ใต้บรรทัด mysql ต้องเพิ่ม event_scheduler=ON
    - ทดลอง restart database เพื่อให้แน่ใจว่า event_scheduler=ON ทำงานได้ โดยใช้คำสั่ง sudo systemctl restart mariadb
    - และเช็คด้วยคำสั่ง show variables like 'event_scheduler';
    
- การใช้ vitest ช่วยในการลดเวลาทดสอบความถูกต้องของข้อมูล และง่ายกว่าการใช้ Sql ในการรัน Test Case 
- vitest สามารถใช้ agent ai ใน anitigravity ช่วยในการ generate test case ที่มีอยู่แล้ว และ code ได้
- vitest ใช้ในการรัน test case โดยต้องผ่าน 100% หากไม่ผ่านต้องกลับไปแก้ procedure หรือ myview(query ที่คำนวนเวลา)
